# CoAgent 核心创新点

**日期：** 2026-06-27  
**来源：** 基于 [coagent-design-spec.md](./coagent-design-spec.md) §1.5 提炼  
**状态：** 摘要文档

---

## 一句话定位

**面向企业的 Agent 运维助手**：Agent 出错或烧钱时，不只告诉你「挂了」，而是从症状推到根因与影响，给出带证据链、把握度和处置边界的方案。

> Agent 出错或烧钱时，90 秒内拿到「该信什么、该做什么、有多把握」——带把握度评分的 Agent 运维助手。

---

## 创新叙事重心

CoAgent 的核心创新不在「告警 +大模型写建议」，而在 **Agent 运维层的可验证处置闭环**。

| 叙事层级 | 评委感知 | 创新性 |
|----------|----------|--------|
| ❌ 弱 | 「又一个 Agent 监控/告警工具」 | 低 |
| ⚠️ 中 | 「大模型帮写运维手册」 | 中 |
| ✅ **CoAgent 主线** | **「Agent 异常 →根因推理 → 把握度评分可验证 → 人工把关处置 → 飞轮复盘」** | **高** |

---

## 五大核心创新

### 1.根因推理，而非日志关联

不是像 Datadog / Azure SRE Agent 那样做基础设施变更关联，而是在 **Agent 运行态** 做处置手册约束下的根因推理：

| 场景 |根因推理 | 处置边界 |
|------|----------|----------|
| **S1 限流** | 流量↑ → 429 → 客服不可用 | 可重试 🟢 |
| **S2 空检索** | 索引滞后 → 空检索 → 幻觉答复 | 不能盲重试 🟡 |
| **S3 超预算** | 发布/流量 → Token↑ → 超预算 | 必须升级止血 🔴 |

差异在 **根因路径**，不在单纯关联。CoAgent 聚焦 **Agent 运行态层**。

### 2. 把握度评分：可验证推理，非大模型自报置信度

三因子公式：

```
总分 = round(100 × (0.35×数据 + 0.35×手册 + 0.30×推理))
```

| 因子 | 含义 |
|------|------|
| **数据** | 数据完备度（字段、日志、工具成功率） |
| **手册** |手册匹配度（症状/类型 vs 运维标签） |
| **推理** | 推理一致性（假设/步骤 vs 错误/日志规则） |

输出 🟢🟡🔴 分级，直接回答 **「敢不敢动」**——针对运维场景 **零幻觉容忍**，把握度评分是主要差异抓手。

| 总分 | 分级 | 动作 |
|------|------|------|
| ≥80 | 可执行 🟢 | 建议按步骤执行（人工确认高风险） |
| 60–79 | 需确认 🟡 | 请负责人确认 |
| <60 | 需升级 🔴 | @值班 / 管理台强调升级 |

### 3. 人工把关的处置边界

评分不是装饰，而是 **动作权限**：

- **S1 🟢**：评分 85，可一键重试 —— **敢动手**
- **S2 🟡**：评分 70，不建议重试，需改知识库 —— **不敢盲动**
- **S3 🔴**：评分 54，高风险步骤不可自动执行，须升级 @ —— **必须升级**

同一类运行失败事件，S1 能重试、S2 不能——评分定义「敢不敢动」的边界。

### 4. 过程可审计 + 反馈飞轮

- 管理台实时推送时间线 + 故障事件全链路留痕
- 👍/👎 反馈驱动运维手册与提示词迭代
- 解决企业 **「说不清当时建议了什么、谁批准的」** 的复盘痛点

价值递进：

```
听见异常 → 获得处置建议 → 评估把握程度 → 团队复盘迭代
   │              │                │              │
 告警+面板      大模型+重试        把握度评分    飞轮+@升级
```

### 5. 赛道切口：填补「Agent 运行态运维」真空

PagerDuty 管基础设施，不管 Agent 运行态；通用聊天机器人不管 **运维决策与置信度**。

| 现成路线 | 做什么 | CoAgent 差异 |
|----------|--------|--------------|
| Azure SRE Agent / Rootly / Harness | 变更 + 遥测 **关联**，根因+置信度 | 不做基础设施变更归因；做 **Agent 运维层根因推理 +评分可验证** |
| Datadog / 通用 应用性能监控 | 告警 → 调查 → 「什么变了」 | 聚焦 **大模型 Agent 失败/成本**，非通用基础设施 |
| 飞书/邮件事件回调通知器 | 挂了提醒 | 有 **推理链 + 评分 + 飞轮**，不是通知 |
| 通用聊天助手 | 自然语言问答 | **事件驱动处置手册+ 结构化处置 + 审计** |

**CoAgent 占用的真空：**  
**「运维 AI 应用本身」+ 「零幻觉可解释处置」** — 与大模型运维观测方向同构，把握度评分是差异抓手。

---

##答辩必讲「三件套」

1. **跨信号 Agent 运维推理** — 不是日志关联，是处置手册约束下的根因链
2. **每步可验证** — 把握度评分分解，评委可追问公式
3. **处置可闭环** — 建议 + 把握度 + 重试/升级，全程留痕

---

## 与竞品一句话（问答备用）

| 他们 | 我们 |
|------|------|
| 管服务器 / 5xx | 管 **Agent 失败、质量、成本** |
| 告警 + 建议 | **评分可验证 + 审计闭环** |
| 云服务 吃数据 | **本地部署，日志不出域** |

---

## 总结

CoAgent 的创新内核：

> **Agent 异常 →根因推理 → 把握度评分可验证 → 人工把关处置 → 飞轮复盘**

同一套代码，叙事从「又一个 Agent 监控工具」上移到 **「带把握度的 Agent 运维处置助手」**——把握度评分三因子 + 三场景（敢动手 → 不敢盲动 → 必须升级）是演示与答辩的主线。
