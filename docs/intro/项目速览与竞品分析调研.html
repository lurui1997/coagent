01 项目速览

核心痛点

  • 静默失败 — 429、空检索，客户先发现
  • 不会处置 — 不知重试 / 改 提示词 / 换模型
  • 成本失控 — Token 飙升，一线无权止血
  • 无法复盘 — 说不清建议了什么、谁批准

目标客户

  • Buyer — 已部署多个 AI Agent 的中小企业 / OPC / 平台团队
  • 角色 — 运维方、AI 架构师、值班工程师
  • 场景 — 客服 Agent、检索增强 问答、内容生成
  • 驱动 — Agent 越多真空越大；金融/政企要私有化

赛道真空

  • PagerDuty 管 基础设施
  • 聊天机器人 管 问答
  • 没人管 Agent 运行态运维

价值递进

听见异常 → 获得处置建议 → 评估把握程度 → 团队复盘迭代

五大核心创新

  1. 1
    根因推理 在 Agent 运行态做处置手册约束推理，非基础设施 日志关联。S1 限流→不可用可重试；S2 空检索→幻觉不可盲重试；S3 Token 突增→须升级止血。
  2. 2
    把握度评分 —可验证推理 总分 = round(100 × (0.35×数据 + 0.35×手册 + 0.30×推理))。数据=完备度 / 手册=匹配度 / 推理=一致性。≥80 🟢 可执行 · 60–79 🟡 需确认 · <60 🔴 需升级。
  3. 3
    人工把关处置边界 评分 不是装饰，而是动作权限：S1 能重试、S2 不能、S3 必须 @升级。同一 run_fail 事件，不同场景处置边界不同。
  4. 4
    过程可审计 + 反馈飞轮 管理台 实时推送时间线 + SQLite 故障事件 全链路留痕；👍/👎 驱动 运维手册与 提示词 迭代。
  5. 5
    赛道切口 — Agent 运行态运维 真空 聚焦大模型 Agent 层（429 / 空检索 / Token），与 Azure / Datadog / 通知器错位竞争；本地部署，日志不出域。

11 SWOT 分析

S · 优势

  • Agent 层切口清晰
  • 把握度评分可追问公式
  • 三场景展示处置边界
  • 私有化 + 飞书
  • 人工把关 符合企业采购心理

W · 劣势

  • 黑客松 MVP，仅 3 场景 mock
  • 无真实 multi-tenant
  • 集成面窄（vs Datadog 2000+ actions）
  • 品牌/客户案例为零

O · 机会

  • AgentOps 2024–2026 快速成熟
  • 大模型运维 观测成熟但 运维决策层真空
  • 国内信创 + 私有化需求
  • 主办方小宿科技 Agent基础设施生态位

T · 威胁

  • Datadog 2025 推 Agent Monitoring + Bits AI SRE
  • Langfuse 加告警后向上延伸
  • Azure SRE Agent GA 占心智
  • 「观测+告警」被误认为已够用

12 竞争策略建议

  1. 错位竞争 — 不正面打 Datadog/Azure，强调「不管服务器，管 Agent 失败/质量/成本决策」
  2. 与 Langfuse/LangSmith 互补 — 「他们是眼睛，CoAgent 是值班大脑」；事件回调 接入现有观测栈
  3. 把握度评分作为 demo 记忆点 — S1 能重试、S2 不能，现场对比最强
  4. 私有化叙事 — 对标美国 云服务 RCA 进不来的金融/政企缝隙
  5. 赛后集成路线 — 接 Langfuse OTel export、PagerDuty escalation、真实重试 API

14 结论

CoAgent 的竞品不是单一产品，而是 三条线的交集空白 —基础设施智能运维 不下沉到 Agent 语义、大模型运维 观测不上浮到处置决策、Gateway 不管「谁拍板」。 把握度评分 +根因推理 + 人工把关 是这个空白区的差异化组合。

最大风险：Datadog / Langfuse 向上/向下延伸 吞噬该空间。 最大机会：Agent 规模化部署后运维真空爆发。
