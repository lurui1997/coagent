## 2. 分版本功能包

### Basic — 当前实现（黑客松 P0）

> **价值主张：** Agent 出错时，90 秒内拿到「该信什么、该做什么、有多把握」。（**出错**：运行失败、质量异常、成本异常等运行态偏离预期，见 [项目介绍](./coagent-project-intro.md)。）

| 模块 | 包含能力 | 验证数据（模拟大模型 E2E） |
|------|----------|-------------------------|
| 接入层 | `/events` + 管理台 触发 + 幂等 | 三场景均可 `status=ok` |
| 推理层 | Router → 3 处置手册 → 大模型 → 评分 | S1 **87🟢** · S2 **72🟡** · S3 **58🔴** |
| 交互层 | 管理台三 Tab + SSE + 回放 + Ultra 面板 | Pipeline ~0–15s（模拟 ~3ms / 真 LLM ~8–15s） |
| 协同层 | 管理台模拟重试 | 重试 200 OK |
| 飞轮层 | 👍/👎 + stats | 标签页四 实时更新 |

### Pro — 企业增强（12h 冲刺 + 赛后）

| 模块 | 12h 计划交付 | 后期扩展 |
|------|-------------|----------|
| 运行时纠偏 | 基于评分/feedback 的 提示词 参数建议 + 可选 webhook 回调修正 | 全自动闭环 + A/B 验证 |
| 模型切换 | `ModelRouter`：按 playbook/分级/成本策略选择模型 | 多厂商路由 + 成本优化器 |
| 企业审计 | 操作人标识 + 审批记录表 + JSON/CSV 导出 | RBAC + SIEM 对接 + 保留策略 |

### Ultra — 智能编排（后期）

| 模块 | 能力描述 |
|------|----------|
| 知识图谱 | Agent、事件、处置手册、处置结果实体关系图；支持多跳根因查询 |
| 历史推演 | 相似 故障事件 检索 + counterfactual 模拟（「若当时降并发评分 会变多少」） |
| 多模式编排 | 按事件复杂度自动选择 Single-Agent / Sub-agent / Multi-Agent / Agent-Team |

## 3. 三场景 × 版本对照

| 场景 | 事件 | Basic 处置边界 | Pro 增强 | Ultra 增强 |
|------|------|-------------|----------|-----------|
| **S1** cs-bot 限流 | run_fail + rate_limit | 评分 82–88 🟢 · 重试 | 纠偏：自动建议降 concurrent；模型切换：429 时 fallback 模型 | 图谱：关联历史限流 故障事件；Team：并发分析 sub-agent |
| **S2** rag-bot 空检索 | run_fail + empty_retrieval | 评分 65–75 🟡 · 禁止重试 | 纠偏：索引 sync 检查 + 提示词 兜底；模型切换：检索失败降级小模型 | 图谱：KB 版本 ↔ 空检索率关联；推演：rebuild 前后对比 |
| **S3** content-bot 超预算 | cost_report + over_budget | 评分 50–58 🔴 · 升级 @ | 模型切换：强制降级 gpt-4o-mini；审计：止血审批链 | 图谱：路由 Token 拓扑；Multi-Agent：成本分析 + 审批 Agent |
