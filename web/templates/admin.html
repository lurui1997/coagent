01 · 业务挑战

Agent 规模化上线，运行态运维缺位

  • 静默失败限流、空检索等异常往往由业务方先感知
  • 处置路径不清重试、改 Prompt、换模型缺乏统一决策依据
  • 成本不可控Token 消耗超预算缺少分级响应机制
  • 执行风险高AI 建议缺乏可验证的置信度评估
  • 审计不完整处置过程、审批人与决策依据难以追溯

标准响应流程

进入处置工作台
  1. ①事件接入 — Webhook 上报或控制台演练注入
  2. ②场景路由 — 按 (type, symptom) 匹配运维手册
  3. ③工具查询 — 拉取指标、配置与手册条目
  4. ④根因推理 — LLM 生成因果链与处置步骤
  5. ⑤把握度评分 — 三因子计算，输出 🟢🟡🔴 分级
  6. ⑥分级处置 — 执行 / 确认 / 升级，同步 IM 通道
  7. ⑦审计归档 — 全链路留痕，反馈驱动手册迭代
